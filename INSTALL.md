# Установка swc-skill-toolkit

Эта инструкция - для сотрудников SWC, которые хотят установить себе скилл-помощник по разработке корпоративных скиллов.

## Шаг 1. Скачать актуальный скилл

Постоянная ссылка на последнюю версию:

**https://github.com/SWCSkill/swc-skill-toolkit/releases/latest/download/swc-skill-toolkit.skill**

## Шаг 2. Загрузить в Claude.ai

1. Открой Claude.ai
2. **Settings** → **Capabilities** → **Skills**
3. **Upload skill**
4. Выбери `swc-skill-toolkit.skill`

## Шаг 3. Разрешить доступ к GitHub

Toolkit читает скиллы и подтягивает скелет через GitHub. Один раз добавь домен в разрешённые:

1. **Settings** → **Capabilities** → **Code execution**
2. **Additional allowed domains**
3. Добавь: `raw.githubusercontent.com`

## Шаг 4. Что делать дальше

Открой новый чат в Claude.ai и скажи, что хочешь сделать. Toolkit автоматически активируется на правильных триггерах:

- «Хочу сделать скилл по [теме]» - запустит сценарий CREATE
- «Обнови мой скилл swc-X» - запустит сценарий UPDATE
- «Расскажи как у вас устроены скиллы» - сценарий CONSULT

## Если что-то не работает

**Toolkit не активируется.** Проверь, что в Settings → Skills он включён.

**Toolkit говорит «не могу прочитать твой скилл».** `raw.githubusercontent.com` не добавлен в allowlist (Шаг 3).

**Toolkit предлагает то, что не подходит твоему случаю.** Это нормально, скажи прямо «не подходит, потому что...» - он пересмотрит. Не пытайся подстроиться под его предложение, если оно не отвечает твоим задачам.

## Обновление

При выходе новой версии toolkit'а:
1. Скачай заново по постоянной ссылке.
2. Settings → Skills → удали старую версию.
3. Загрузи новую через Upload skill.

История версий: [CHANGELOG.md](./CHANGELOG.md)
