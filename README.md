# swc-skill-toolkit

Инструментарий для создания и поддержки корпоративных скиллов SWC.

## Назначение

Скиллы SWC построены по единому архитектурному паттерну:
- репозиторий на GitHub (под `SWCSkill/`)
- разделение «контент в репо / поведение в скилле»
- сборка в `.skill` через GitHub Action
- постоянная ссылка через Releases для распространения

`swc-skill-toolkit` собирает в одном месте всё, что нужно, чтобы
делать новые скиллы по этой же архитектуре - и поддерживать существующие
без расхождений и копипасты.

## Что планируется внутри

- **Шаблон-скелет репозитория** (`skill-template/`) - готовая структура
  под `Use this template` в GitHub. Создаёшь новый репо - получаешь рабочий
  каркас за 30 секунд.
- **Универсальные скрипты:**
  - `build_skill.py` - собирает `.skill`-архив из шаблона + референсов.
  - валидация YAML frontmatter, проверка целостности ссылок, синхронизация
    `VERSION` ↔ `CHANGELOG`.
- **Конвенции и документация:**
  - как структурировать SKILL.md (роль / архитектура / рабочий цикл /
    диагностика / соседи).
  - принципы выбора границы «что в скилле, что в репо».
  - правила нейминга репо и скиллов.
- **Сам скилл** `swc-skill-toolkit` для использования в Claude Code -
  помогает разработчику править SKILL.md, валидировать, генерировать
  changelog, бампать версии.

## Зачем

Когда у тебя один скилл - можно делать вручную. Когда их три-четыре,
становится больно дублировать настройки и ловить расхождения в архитектуре.
`swc-skill-toolkit` снимает эту боль и фиксирует «как у нас правильно»
в одном месте.

## Существующие скиллы SWC, использующие эту архитектуру

- [swc-assistant](https://github.com/SWCSkill/swc-knowledge-base) -
  продукты, бренд, партнёрка
- [swc-manajet-expert](https://github.com/SWCSkill/manajet-knowledge-base) -
  работа в ManaJet через интерфейс
- [swc-orgpolicies-guru](https://github.com/SWCSkill/swc-orgpolicies-guru)
  (планируется) - орг.политика и регламенты SWC
- [manajet-api-helper](https://github.com/SWCSkill/manajet-api-helper)
  (планируется) - API ManaJet для разработчиков

## Статус

🚧 **В разработке.** Будет собран после того, как `swc-orgpolicies-guru`
будет сделан вручную - на втором скилле проявится, что именно стоит
вынести в общий toolkit, а что оставить специфичным для скилла.
