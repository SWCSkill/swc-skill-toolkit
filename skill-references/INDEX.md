# Карта референсов swc-skill-toolkit

Содержание `references/` - что где смотреть.

## Сценарные гиды

- **`workflow_create.md`** - пошаговый гид для сценария CREATE (создание нового скилла с нуля). Открывай, когда сотрудник пришёл с задачей «хочу сделать скилл» или эквивалентной формулировкой.
- **`workflow_update.md`** - пошаговый гид для сценария UPDATE (обновление существующего скилла). Открывай, когда сотрудник называет имя существующего скилла и описывает изменение.

## Принципы и правила

- **`architecture.md`** - принципы архитектуры скиллов SWC. Читай при обсуждении «почему так устроено» и при проектировании нового скилла. Объясняй из этого файла, когда сотрудник сомневается «можно ли иначе».
- **`conventions.md`** - конкретные правила: имена, структура, версионирование, теги. Используй при сборке скелета и при проверке имени.
- **`changelog_style.md`** - формат и правила CHANGELOG.md. Используй при подготовке записи в CHANGELOG для новой версии.
- **`checklists.md`** - чек-листы валидации. Прогоняй перед отдачей архива и после заливки.

## Как пользоваться

При входе в сценарий - подгружай соответствующий workflow и веди по нему. По ходу - подсвечивай конкретные правила из `conventions.md`, объясняй принципы из `architecture.md` если возникают сомнения.

Не пытайся прочитать всё сразу - подгружай по релевантности к текущему шагу.
